# adia-ui-factory v0.2.0 — scope re-carve (2026-06-03)

A grounded re-carve after a comprehensive read of how chat-ui builds and dogfoods adia-ui (6 parallel research passes over `apps/`, `playgrounds/`, `packages/web-modules/`, `.agents/skills/`, migrations). The v0.1.0 six-skill design was too thin to _manage real adia-ui projects_. This locks the v0.2.0 roster.

## What the research established

**Project shapes (5).** Apps cluster into single-surface (tasks), heterogeneous rollup (genui), homogeneous rollup (saas), page-DUO-dominant (user-flow), shared-foundation multi-surface (embedded-app) — over a consistent **four-axis layout** (`spec/ plan/ app/ skills/`) with the **page-trio** (`.html` + `.contents.html` + `.contents.js`, ADR-0021) vs **page-DUO** (no `.contents.js`, for declarative pages) distinction.

**Shells (4 shipped + 1 forthcoming).** `admin-shell` (12-element cluster), `chat-shell` (7), `editor-shell` (7), `simple-shell` (3). **`verse` is NOT a shell** — it's a typographic register (`[verse]`, sibling to `[prose]`/regular; toggled via `<theme-panel register>`), so it lives in theming. **`adia-embed-shell` is forthcoming** — the official promotion of the embedded-app pattern (light-DOM custom element, host sizes/centers via absolute positioning, DataClient/projection, content-less `router-ui` with query-param URL management). Shared shell conventions: bespoke vocabulary (ADR-0023/0024), slot semantics, state-as-attribute, cluster-barrel registration (`@adia-ai/web-modules/{shell,chat,editor,runtime}`), behavior-only hosting.

**Authoring workflow.** Orient → Plan (§SpecToUi: 19-rung ladder + wireframe rubric) → Scaffold → Compose → Wire → Verify → Migrate/Ship, with real gates: `audit:shell-composition`, `audit:native-primitive-leak`, the dogfood-sweep probes, the browser probe.

**Data/state (consolidated).** Signals (`signal`/`effect`), Service/Controller/Command (tasks), DataClient/projection + mappers (embedded-app), property-API binding (saas), declarative `data-*` (user-flow). Plus hybrid topologies — SPA islands in SSR-hydrated pages.

**Gen-UI / a2ui (consumer side).** The `runtime` cluster (`a2ui-root`, `gen-root`), `registerResolver` (from `@adia-ai/web-modules/generative`), `generate_ui` + `refine_ui` via the MCP, and corpus options (the core 284-chunk training data vs. a custom corpus).

**Migration (rich).** 5 types (version upgrade · port-to-adia · SPA↔SSR · additive · tier-extraction); documented breaking history (0.0.20 ten-item, 0.4.0 shell retirement, token renames); a 5-step discipline (read MIGRATION GUIDE → `git grep` audit → `perl` sweeps → verify gates → report); codemods; MCP aids (`search_chunks`, `check_anti_patterns`, `convert_html`).

**Consumer/maintainer split (sharp).** Consumer (THIS plugin): `adia-ui-kit` (compose), `adia-ui-gen-review`, `dogfood-sweep`, `adia-ui-migration`. Maintainer (plugin 2 = **`adia-ui-forge`**): `adia-ui-authoring`, `adia-ui-a2ui`, `adia-ui-release`, `adia-ui-ops`.

## Tooling

- **`bin/adia-scaffold`** (extend) — emit the real project shapes: single-surface, page-trio & page-DUO, rollup, shared-foundation; four-axis dirs; per-shell starters (admin/chat/editor/simple, +embed when it ships); SSR integration layer.
- **`bin/adia-lint`** (extend) — add the consumer audits: native-primitive-leak (`<button>`→`<button-ui>`), shell-composition smells (e.g. raw `<header>` in `admin-page-header`, missing `[data-resize]`), legacy-shape detection (retired ADR-0024 attributes). Still advisory, never-blocks.
- **Hook** — runs the enriched lint on writes.

## Phased build plan

- **(1)** Re-carve foundation — revise `adia-ui-factory`, add `adia-ui-project`, extend `bin/adia-scaffold` for shapes.
- **(2)** `adia-ui-shells` (+ embed forthcoming) + extend `bin/adia-lint` (native-primitive-leak, shell-composition).
- **(3)** `adia-ui-data` (consolidated) + reconcile `spa`/`ssr` for hybrid.
- **(4)** `adia-ui-genui` + `adia-ui-llm` (JIT research: a2ui runtime + corpus).
- **(5)** `adia-ui-verify` (enriched) + `adia-ui-migrate`.
- **(6)** Validate + full 9-critic red-team; fold findings.

Each phase committed + gated. Thin spots get just-in-time research before authoring: the a2ui consumer runtime + corpus (genui), the embed-shell pattern (shells), hybrid hydration/section-registration (data).

## Trade-offs

- **P6 (context economy)** — 11 always-on skill descriptions + the ~24-tool MCP is a real cost; mitigated by consolidation (shells→1, data→1), tight descriptions, and progressive disclosure (depth in references). This is the comprehensiveness↔cost trade the scope requires; the red-team will re-check it.
- **Forward-compat** — `adia-embed-shell` and the `verse` showcase are documented as emerging; re-bake when they ship (paired with the pinned MCP snapshot).

## Out of scope → plugin 2 (`adia-ui-forge`)

Authoring the framework itself: new library primitives/traits/shells, the a2ui pipeline internals, git releases (9-package lockstep), training-data/corpus curation, ops. Built later, same carve discipline.
