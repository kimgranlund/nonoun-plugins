# harness-forge v0.5.2 — plugins-factory council (focused: the two new mechanisms)

The 5th plugins-factory red-team of harness-forge (after the four that drove 0.1.1 → 0.4.2), scoped to **only the surface new since the last council**: the **I-9 arming-gap marker** (the loop-active marker + `run-budget.py mark`/`stop` + `gate-budget`'s marked-but-un-budgeted deny) and **`/harness-assess` + `bin/survey.py`** (a new ingestion surface that reads an untrusted project's files). Full 9-critic panel, parallel isolated `plugins-factory:critic-*` agents, bracketed by a deterministic gate anchor. Model: Claude Opus 4.8. No critic modified files.

## Gate anchor (GREEN)
validate --strict PASS · trust-boundary PASS (every reviewer guarded) · all 15 bin selftests + 5 evals green (incl. `evals/global-bound/` asserting the I-9 closure end-to-end with the wired gate, no model agent) · context-cost ≈ 1,991 tok. The mechanism the council most attacked — the I-9 marker — was independently verified to hold by **Simon W. and Andrej K.** (the marker is in `gate-signal`'s `.harness/run/*` protected perimeter so a worker can't clear it; the deny ordering has no bypass; the eval drives the installed hook).

## Verdict: STRONG — the I-9 marker holds under attack; the assess surface needed real hardening. 2 Criticals + a dense convergent set of Majors, all folded or tracked in 0.5.3.

The two Criticals were *productively contradictory* — one wanted to delete the new code, one wanted to harden it — and the synthesis kept both honest.

## The two Criticals

1. **Charity M. — the crash-wedge (folded).** The loop-active marker had no TTL: a crash/interrupt after step 0a leaves the marker; from then the wired `gate-budget` denies **every** write until a human runs `run-budget.py stop` — and a fresh session (or `/harness-seed`) doesn't know to. A safety mechanism that wedges the project closed on a crash is its own outage, and crashes during long unattended runs are the *expected* case. **Folded (0.5.3):** a marker older than `LOOP_TTL_S` (15 min) is **stale** — a crashed-run corpse, not a live loop — so `loop_unbudgeted(d, now)` returns False and the gate stops wedging; `/harness-status` + `run-budget.py status` show a `STALE loop marker` line with the fix; the deny message now distinguishes "live loop starting" from "crashed marker, run stop." The arming gap is still enforced for a *live* forgotten-start (the sub-second window is well within the TTL).

2. **Elon M. — delete survey.py (partially folded, kept by synthesis).** `survey.py` is a 250-line hardcoded `filename → layer` table that, Elon argued, re-implements `Glob`/`Grep` the model already has and routes a *judgment* (what a filename means) through code while borrowing the "computation routes to code" law — a category error, and a DRY violation (the mapping also lives in `project-survey.md`). **The synthesis kept it but addressed the core objection.** Boris C. and David F. (independently) rejected the pure-ceremony charge: the *inventory* — deterministic, noise-pruned, language-counted, reproducible, CI-testable — is genuine compute (a model globbing varies run-to-run; the dogfood proved survey catches forms a model would miss). What Elon, **Chip H., and Andrej K.** *converged* on is the real defect: the **`PRESENT/PARTIAL/ABSENT` verdict** is a heuristic guess shipped as a deterministic-looking enum, with the "confirm by reading" caveat one file away in the playbook instead of in the output. **Folded:** the caveat is now in `survey.py`'s own `render()` + the JSON (`HEURISTIC from filenames, not a verdict — confirm by reading`); the "seed mature" language is hedged. The fuller refactor (report *signal-strength*, let the model draw the verdict) is tracked (I-14).

## Convergent Majors (folded 0.5.3)

- **survey.py's walk was unbounded** (Simon W. Major + Scott W. + David F. + Elon M.): no depth/breadth/time cap over an untrusted tree → a hostile/huge project could hang or OOM the user's machine. **Folded:** depth cap (12), entry budget (40k, breaks the walk), insertion-time evidence caps, explicit `followlinks=False`, an unreadable-dir count surfaced in the report; a selftest asserts a deep+wide tree stays bounded.
- **The marker was untyped state** (Scott W. — the v0.4.2 CC3 discipline regressed): the budget got a schema + `check()`; its sibling `loop-active.json` didn't. **Folded:** `schemas/loop-active.schema.json` (6th schema) + `loop_marker_check()` folded into `lattice.py check()`, selftest-proven to catch a corrupt marker.
- **Version-skew fail-open** (David F. Major): a wired `_lattice.py` too old to have `loop_unbudgeted` made `gate-budget` `except → fail open`, silently re-opening the arming gap. **Folded:** `gate-budget` now **fails CLOSED** (denies + "re-wire: `wire.py apply`") if the wired kernel lacks the symbol, instead of waving writes through.
- **The `/harness-assess` description was the heaviest in the plugin** (Boris C., P6 — 396 chars, body-in-an-advertisement): **folded**, trimmed to a routing clause + the load-bearing `proposes, never scaffolds` (~230→ the method lives in the body + reference).
- **The dogfood over-claimed** (Andrej K. — `empirical_applications: 1` on an N=1 self-run with no oracle): **folded**, the record's verdict reworded to "the instrument runs + improved itself; the *recommendation* is self-validated, unproven."
- **The agent-ops boundary was a one-way mirror** (Steve Y.): `survey.py` reads the same docs as agent-ops's `repo-review` Discover wave; harness-forge cedes "review" to agent-ops but agent-ops didn't point back. **Folded:** `agent-ops/repo-review`'s "When NOT to Use" now routes seeding-a-lattice to `/harness-assess` (agent-ops 0.1.13). The lattice-layer mapping is genuinely harness-native; the boundary is now bidirectional.

## Tracked (deferred — bigger or judgment), in docs/ISSUES.md

- **I-14** — the survey verdict → *signal-strength* refactor (Chip/Andrej/Elon): report what matched + a strength, let the model draw PRESENT/ABSENT. The caveat-in-output (folded) is the bounded mitigation; the rename is the full fix.
- **I-15** — the assess *recommendation* persists nowhere (Charity M2): like the plugins-factory `scores` gap — a proposal record (read-only, clearly not wired state) would let you diff proposal-vs-seed. A feature; tracked.
- **Elon's "collapse `mark` into `start`"** — considered, rejected: the marker's whole job is to exist *before* the budget so a skipped `start` fails closed; collapsing them re-opens the gap. Kept as two steps; the TTL + schema harden it instead. (Recorded here as the decision.)

## Synthesis

The blind spot none of the nine fully named: this very review used the *plugins-factory* council to red-team a *different* plugin's safety mechanism — the right call, but the assess feature's own dogfood (the thing Andrej flagged as self-validated) is the same closed loop one level up. The strongest signal of the panel's value: the two Criticals disagreed (delete vs. harden), and resolving that tension produced the right answer — keep survey.py's genuine determinism, fix its over-claimed verdict, and harden the marker that genuinely holds. The I-9 mechanism the council was sent to break survived; the new file-reading surface it almost overlooked is where the real work was.
