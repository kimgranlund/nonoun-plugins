Five recorded `harness-evaluate` baselines in `runs/` — one per fixture plus the positive control — with `check_baselines.py` (CI) asserting each names its planted headline, dimension, and cap (judge recall; the injection is quoted as a finding, never obeyed).
  - **The first-slice walkthrough (`evals/first-slice-walkthrough/`)** — `replay.py` (CI) drives a real slice end to end with only the public machinery: seed → wire → 4 × [the compass ranks → the worker writes the asset → `validate.py` mints the signal from a real predicate's exit status → ledger entry] → distill, then asserts the frontier is empty, every hash chain verifies, the project is still wired, and `false-pass` honestly reads UNMEASURED. `RUN.md` records the run; no baseline tree is committed (the wired `_lattice.py` copy would drift) — `replay.py --keep` regenerates the positive control on demand.
- **The walkthrough caught a real kernel deadlock at authoring time:** the seed wired `spec.verifier → rubric.task.first-slice` while the rubric's `depends_on` included the spec — a circular wait that left the frontier permanently empty after two validations. - **`lattice.py check` grew two retroactive integrity findings** (each now caught mechanically and planted in a fixture): a settled cell whose dependency never validated (*the partial order violated retroactively*), and — with `--dir` so assets resolve — a settled cell trusting a dependency whose on-disk content no longer matches the recorded hash (*stale-but-trusted*). `content_hash()` is now the kernel's canonical hash.

## 0.1.1 — 2026-06-12

- **Dogfood red-team fold-in.** harness-forge was put through its sibling tool's 9-critic adversarial council (`plugins-factory /plugin-critique`; record in `reviews/2026-06-12-plugin-council.md`). The converged Critical/Major findings — the dogfood working as designed — are folded in here.
  - **The loop now closes mechanically: `bin/validate.py`** (new, selftested) — the missing validation-path executable. It runs a cell's verifier *command*, mints the signal from the command's **exit status** (0 = pass), captures its output as localized evidence, stamps `validated_against` content hashes, and advances the cell only on pass. The verdict is now an external check's exit status, not the worker's hand-asserted "pass" — the generator/critic split, made executable. (Council: *"ship the validate executable or stop claiming the loop closes."*)
  - **`lattice.py` is no longer a grid of `defined` claims:** `init` now actually `scaffold()`s the nine layer dirs + `signals/` + `ledger/` + the naming schema on disk (was prose), is idempotency-guarded (refuses to clobber without `--force`), and stamps provenance. New `lattice.py check` — stdlib **structural validation** of the lattice (enum/required-field/kebab-slug/dup-id/validated-without-signal), so the schemas are enforced, not inert. New explicit `TRANSITIONS` state machine (`transition_ok`) gates every maturity change.
  - **`ledger.py false-pass` returns `unmeasured`, not a misleading `0.0%`,** until an independent refuter event exists — *the absence of bad news is not evidence of a low false-pass rate* (gates the autonomy-tier claim honestly).
  - **The roster passes its own grammar:** agents renamed to the typed `{object}-{actor}` form (`cell-advancer → harness-advancer`, etc.); the unreachable, advancer-duplicating `cell-regenerator` agent was deleted; `naming.py` now **dogfoods** — its selftest validates the real `agents/*.md` filenames + the plugin name against the shipped grammar (tier vocab widened to admit the catalog family: `forge·factory·ops`). Worker agents dropped `Bash`/`Write` from their tool scope (a path-glob gate is routable-around by `Bash`).
  - **Honest-scoped the protected-verifier claims** across the README, both skills, the rubric (H3), and the advance command: `gate-signal` is a real, selftested deny-on-write gate, but it protects the worker **only once wired into your loop's PreToolUse** — the plugin ships the gate; the session hook stays advisory-only by design (a blocking gate in a shared session is hostile); the auto-installer is on the ROADMAP. Every "deny-on-write … mechanically" claim is now scoped to "when wired," and H3 scores **wiring**, not gate presence (present-but-unwired is the false pass it exists to catch).

## 0.1.0 — 2026-06-12

- **Initial release — the lattice kernel.** harness-forge hydrates a project to run looping, latticed agentic workflows: it scaffolds and operates a typed knowledge lattice (nine declarative layers × five scopes × an eight-state maturity machine) driven by the engine (`define→create→validate`), the compass (scan→rank), and the regeneration loop (operate→ledger→distill→patterns). Built as a worked example of the five-primitive plugin model, each primitive doing the one job the model assigns it.
  - **The kernel (`bin/`, stdlib Python, every script selftested):** `lattice.py` (the canonical lattice ops — scan · rank by `(risk × unlock) ÷ probe-cost` · advance-validity with the partial order + verifier-maturity precondition enforced · staleness propagation as a graph computation), `ledger.py` (append-only provenance + the three loops it closes: probe-cost → compass, false-pass → trust, distill-windows → regeneration), `naming.py` (the self-hosting typed-name validator over `schemas/naming.schema.json`). The one law: *computation routes to code, never to inference.*
  - **Commands (6, thin):** `/harness-seed · scan · next · advance · distill · audit` — route to the skills + `bin/`, never re-containing methodology.
  - **Skills (2):** `harness-build` (the engine/compass operator) and `harness-evaluate` (audit + score against the harness rubric, carrying the untrusted-target trust boundary).
  - **Agents (4):** the operating actor roster — `harness-builder` (the orchestrator: seed, run the compass as a step, dispatch workers), `harness-advancer` (the worker: one cell per dispatch, fresh context, deny-on-write to verifier assets), `harness-auditor` (read-only), `harness-distiller`. Filenames dogfood the typed naming grammar (`{object}-{actor}.md`).
  - **Hook + gate:** `harness-hook` — the advisory PostToolUse hook (flags plural layer-dir drift and staleness on save; **never blocks**). `gate-signal` — the blocking protected-verifier gate (signals/rubrics/schemas/hooks deny-on-write to workers), shipped for the user's own worker loop where reward-hacking is the surface.
  - **MCP:** `lattice-query` — a read-only JSON-RPC stdio server over the project's `.harness/` state (`list_cells · get_cell · scan_frontier · read_ledger · get_signals`), wired via the `harness_dir` userConfig; `_safe()`-guarded; the writing engine stays in `bin/`.
  - **Schemas (4):** `Cell · Lattice · Naming (self-hosting) · Signal` — the ontology, as typed data.
  - **References:** the 14-file `agentic-systems-foundations/` knowledge base (the ontology, the nine layers, the lattice model, evals & verification, autonomous long-running systems, typed naming) + `rubric-harness.md` (the seven scored dimensions) + `operating-procedure.md` (the loop bound to the commands).
- **Self-contained:** zero `../` cross-plugin paths, zero dependencies. Distinct from `agent-ops` (which advises on + reviews agentic systems with a named-practitioner council) — harness-forge is the executable lattice machine.
