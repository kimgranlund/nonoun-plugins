# Changelog

## 0.4.0 — 2026-06-13

- **The bounded autonomous loop — the v0.4 capstone.** `/harness-run` ships the loop the plugin was always *about*: the **`harness-builder` orchestrator** (restored — its 0.3.1 deletion was explicitly "returns with `/harness-run`") advances the frontier automatically under **hard caps** (max-cells · max-iterations · wall-clock), and halts with a report. It is **bounded and stoppable by construction**, **attended by design** (surfaces at every stop, never fire-and-forget, never raises its own caps), and refuses to claim autonomy it hasn't earned (run attended until `ledger.py false-pass` shows a track record).
- **The budget/no-progress stop-gate is now WIRED, not asked** — closing the 0.3.1 honest-scope debt (Chip H.'s CV4) and the positive-control baseline's H5=2. Three independent code mechanisms halt a runaway loop: `ledger.py no-progress` **detects** a cell whose last N validates all failed (0.3.1); the orchestrator **blocks** it (`lattice.py block`, a protected write — new); and the wired **`gate-budget`** hook (new, selftested) **denies** any further Write/Edit to a blocked cell (PreToolUse, exit 2), so the circuit breaker holds even if the worker ignores the instruction to stop. A blocked cell drops out of `rank` and `advance-validity`; `gate-budget` is installed by `wire.py apply` alongside the other gates.
- **`evals/stop-gate/`** (new, CI-replayed) — proves the breaker breaks end to end with only public machinery (no model agent): a capability cell fails a real verifier 3×, `no-progress` fires, the cell is blocked, and the before/after control shows it go from rankable-and-writable to unrankable-and-denied (by both `rank` and the installed `gate-budget` hook).
- **Kernel:** `lattice.py block`/`unblock <cell>` + `set_blocked()`; `rank` now excludes blocked cells; `blocked_reason` added to `cell.schema.json` (so it passes the 0.3.1 unknown-key check). `KERNEL_VERSION` → `0.4.0` (the wire-drift gate tracks it). Roster back to **12 agents**, commands to **8**.

## 0.1.0 — 2026-06-12

- **Initial release — the lattice kernel.** harness-forge hydrates a project to run looping, latticed agentic workflows: it scaffolds and operates a typed knowledge lattice (nine declarative layers × five scopes × an eight-state maturity machine) driven by the engine (`define→create→validate`), the compass (scan→rank), and the regeneration loop (operate→ledger→distill→patterns). Built as a worked example of the five-primitive plugin model, each primitive doing the one job the model assigns it.
  - **The kernel (`bin/`, stdlib Python, every script selftested):** `lattice.py` (the canonical lattice ops — scan · rank by `(risk × unlock) ÷ probe-cost` · advance-validity with the partial order + verifier-maturity precondition enforced · staleness propagation as a graph computation), `ledger.py` (append-only provenance + the three loops it closes: probe-cost → compass, false-pass → trust, distill-windows → regeneration), `naming.py` (the self-hosting typed-name validator over `schemas/naming.schema.json`). The one law: *computation routes to code, never to inference.*
  - **Commands (6, thin):** `/harness-seed · scan · next · advance · distill · audit` — route to the skills + `bin/`, never re-containing methodology.
  - **Skills (2):** `harness-build` (the engine/compass operator) and `harness-evaluate` (audit + score against the harness rubric, carrying the untrusted-target trust boundary).
  - **Agents (4):** the operating actor roster — `harness-builder` (the orchestrator: seed, run the compass as a step, dispatch workers), `harness-advancer` (the worker: one cell per dispatch, fresh context, deny-on-write to verifier assets), `harness-auditor` (read-only), `harness-distiller`. Filenames dogfood the typed naming grammar (`{object}-{actor}.md`).
  - **Hook + gate:** `harness-hook` — the advisory PostToolUse hook (flags plural layer-dir drift and staleness on save; **never blocks**). `gate-signal` — the blocking protected-verifier gate (signals/rubrics/schemas/hooks deny-on-write to workers), shipped for the user's own worker loop where reward-hacking is the surface.
  - **MCP:** `lattice-query` — a read-only JSON-RPC stdio server over the project's `.harness/` state (`list_cells · get_cell · scan_frontier · read_ledger · get_signals`), wired via the `harness_dir` userConfig; `_safe()`-guarded; the writing engine stays in `bin/`.
  - **Schemas (4):** `Cell · Lattice · Naming (self-hosting) · Signal` — the ontology, as typed data.
  - **References:** the 14-file `agentic-systems-foundations/` knowledge base (the ontology, the nine layers, the lattice model, evals & verification, autonomous long-running systems, typed naming) + `rubric-harness.md` (the seven scored dimensions) + `operating-procedure.md` (the loop bound to the commands).
- **Self-contained:** zero `../` cross-plugin paths, zero dependencies. Distinct from `agent-ops` (which advises on + reviews agentic systems with a named-practitioner council) — harness-forge is the executable lattice machine.
