## Folded before 0.2.0

1. **Provenance gate skipped the v0.2 surface** (Karpathy **Critical**; Charity/Huyen/Farley converged). `check-sourcing.py`'s `LIBRARY_SKILLS` was a hard-coded 4-skill tuple — it never scanned `product-architecture` or `product-operations`, so ~33 new reference files were unchecked while the CHANGELOG claimed coverage. **Fix: rewrote the gate to DERIVE its scope from the skill tree** (every `skills/*/references/**/*.md`, excluding `rubrics/` which carry no frontmatter by design) so no future skill can silently fall out. Re-ran: **0 gaps across 134 refs** including all 20 architecture + 13 operations files — proving the wave authors' frontmatter discipline actually held, and making the CHANGELOG claim true.
2. **Critic count "23 vs 22"** (Elon/Wlaschin/Karpathy Major). **This was a panel miscount** — ground-truth `ls agents/critic-*.md` returns **23** files (17 v0.1 + 6 v0.2), and `--strict` (which requires agent `name:`) passes on all of them; every doc correctly says 23. **Fix: mechanized it anyway** — added a roster-count assertion to `check-sourcing.py` (the council's stated "Roster (N critics)" must equal the critic-file count), so the number is now a verified property and a future miscount (by a human or a critic) is caught.
3. **One-directional content/IA hand-off + always-on description tax** (Steve/Boris Major). `product-architecture` carried the "system vs. reusable unit" boundary but `product-patterns` had no reciprocal pointer, so a request landing in patterns first never learned architecture existed. **Fix: added the reciprocal "Boundary with product-architecture" block to `product-patterns/SKILL.md`**, and trimmed the manifest description (~halved) + both new SKILL descriptions to trigger-set-plus-boundary (paying down P6).
