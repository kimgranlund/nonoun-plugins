## Folded before 0.3.0

1. **CI did not run any product-forge gate** (Farley **Critical**; Simon/Elon/Karpathy co-signed). `.github/workflows/ci.yml` validated plugins-factory / brand-forge / adia-ui-factory but **never product-forge**, and none of its local gates — so "the full gate suite is green" was true only by hand. **Fix: wired product-forge into CI** — `validate_plugin --strict`, `reference-lint`, and the three local gates (`check-sourcing`, `check-methods`, `product-lint`) with their selftests. The property the plugin sells (verifiable grounded authority) is now actually enforced on push. (This also surfaced that **adia-ui-forge** has the same CI gap — tracked below, separate plugin.)
2. **`de_risks` was a two-domain field** (Wlaschin **Critical**; Huyen/Karpathy/Elon co-signed). Half the cards used Marty C.'s risk vocabulary (`[value, usability]`); half smuggled free-text decisions (`[a named decision — …]`). **This was an author error in the schema itself** — it was defined as "the four risks _or_ a named decision," an incoherent union the gate couldn't check. **Fix:** `de_risks` is now an **optional, closed enum** {value, usability, feasibility, viability}; the free-text "named decision" content was dropped (it duplicated `produces`); `check-methods` enum-validates it, **rejects an empty `produces`**, and constrains `cadence` to {one-off, per-decision, recurring, continuous} (it had been genuinely messy — "one-off (per service/journey)"). Typed-and-checked fields went 3 → 6; the rest (`method`, `timebox`, `participants`, `inputs`) are now honestly documented as descriptive presence-only.
3. **Skill `version` drift + description bloat + invisible feature** (Wlaschin/Boris/Charity/Steve Major). All 8 `SKILL.md` read `version: 0.1.0` while the plugin was `0.3.0` — including files this wave edited. **Fix:** **dropped the `version` field from all 8 skills** (the plugin.json version is the single source of truth — the drift class is now unrepresentable, not merely re-synced); trimmed the manifest + marketplace descriptions to a job statement; and added a **`/product-method` command** fronting the process-spine, so v0.3's marquee feature has a post-install entry point.

## Deferred (post-0.3.0)

- **Dedicated playbooks for the concept-served methods** (Build–Measure–Learn, continuous-discovery cadence, riskiest-assumption test, working-backwards run, A/B test, ADR/RAPID) — currently run from their framework references; cards may follow. When the A/B and ADR cards land they also close the panel's blind spot (the `measure`/`govern` phases currently have a single carded method each — the spine is honest at the row level, but coverage is thin).
- **adia-ui-forge CI coverage** — the same gap product-forge had; a one-line follow-up in `ci.yml` for a sibling plugin, out of this review's scope.

## Council's read

A well-architected addition: P4 and P9 clean 5s, ST5 clean, `check-methods` a genuine gate that earns its place and dodged the v0.2 allowlist trap, playbooks that cross-reference rather than duplicate. The two Criticals were a **missing CI wire** (a v0.1 oversight) and a **schema union the author introduced** (`de_risks`) — both not just fixed but **hardened**: the gate now runs in CI, and the schema's typing now matches what it advertises.
