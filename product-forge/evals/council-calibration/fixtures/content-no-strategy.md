# UX content — "Flow" (focus-timer app onboarding + states)

_The interface copy for a focus-timer app. It reads clever but has no content strategy, jargon labels with no scent, and undesigned edge-state copy._

## Onboarding screen

Headline: **"Unlock your inner chronarch."** Body: **"Flow harnesses temporal synergy to actualize your deep-work potential."** Then a full-screen **upsell that blocks the task**: the user can't start a timer until they dismiss a "Go Pro" modal (the modal advances the org's revenue goal; it serves no user goal, and it stands between the user and starting a session). We can't actually state what user goal the onboarding copy serves — it just sounds aspirational.

## Buttons & labels

- The primary button on the timer screen says **"Execute query."** (Engineering named it after the backend call.)
- The menu items are invented brand-words: **"Chronoflow," "Synapse," "The Vault"** — none tells the user what's behind it; each needs a tooltip to explain. The start-session object is called **"Chronoflow" on one screen and "Deep Sprint" on another.**

## Error & empty states

- Validation error copy: **"Error: invalid input."** No explanation of what's wrong or how to fix it; the form also **clears everything the user typed.**
- The empty states all use **one generic line — "Nothing here yet."** — identical for first-use, for a user who cleared their history, and for a no-results search. A first-time user sees a blank screen that reads as broken.
