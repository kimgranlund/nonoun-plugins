## 0.2.36 — 2026-06-13

- **I-12 (self-red-team, Simon W. Critical) — the executing gates' trust boundary is now a code interlock, not a docstring.** `check-mcp-liveness.py` and `context-cost.py --with-mcp` spawn a target plugin's MCP server (its `.mcp.json` `command`/`args` run as a subprocess with the full ambient environment) — so against a malicious bundle the "server" is arbitrary code execution, the lethal trifecta this plugin grades Score-1 in others. The guard *was* a comment ("TRUSTED catalog plugins only"); prose is not a mechanism. Now the executing paths **refuse by default** (exit 3, spawning nothing) and require an explicit per-invocation trust assertion: the **`--trusted-source`** flag OR **`PLUGINS_FACTORY_TRUST_EXEC=1`** in the env (CI sets the flag; a human almost never does by accident). The interlock is **narrow** — `context-cost.py`'s default static audit executes nothing and stays open with no flag; only the opt-in `--with-mcp` path is gated. Both gates' `selftest`s now prove the interlock (refuse-without-trust → exit 3; flag/env authorizes; static unaffected). CI updated to pass `--trusted-source` on its two executing invocations; CLAUDE.md's runnable examples + the gate descriptions updated. `safe state is the default; unsafety requires a deliberate, visible act` — the inverse of before.

Ships a `selftest` that proves the measurement (incl. folded `>` block scalars — the dominant description format here, which a naïve single-line parse silently undercounts 10×), the WARN/FAIL tiers, and a `--budget` breach. Wired into CI (`selftest` + `marketplace .`).
