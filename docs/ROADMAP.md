## Track 5 — integration & host-portability (new direction, 2026-06-14)

The vision (Kim): a **standardized, hardened integration contract** every catalog plugin conforms to — host-harness-level + agent-level — guaranteeing reliable integration while staying token-efficient, *and* aware of which host it runs in (Claude Code · Codex · OpenCode · Gemini CLI · Pi · Hermes · …). To design + sequence; the realistic shape, with what's already true:

- **Already portable — the MCP layer.** MCP is a cross-host standard, so `lattice-query` · `repo-memory` · `brand-corpus` · `product-corpus` already serve any MCP-speaking host (Claude Code, Codex, Gemini, …). The portable core exists; this track *leans on it* as the host-neutral spine.
- **Already a router — commands → skills/agents.** The five-primitive model *is* an entry/router: thin typed command entry points that classify + route, never re-containing the body (loaded on demand), with `context-cost.py` enforcing the token budget. The work is to **promote convention → contract** — a validated "integration manifest" every plugin passes (a gate), so reliable integration is *checked*, not disciplined — and to weigh a more aggressive single-router-per-plugin against the discoverability cost of hiding primitive descriptions from the model at session start (the eager-description trade-off).
- **Host-specific — install/wire + the markdown primitives.** `commands/agents/skills/hooks` are Claude-Code-format; they won't load in another host without that host's format. Near-term host-awareness = a deterministic **`host-detect`** (env / config-dir probe → host id + confidence, in the *computation-routes-to-code* style) + a per-host **adapter** for the host-specific bits (where `wire.py` writes config, the hook shape). Full primitive portability (re-expressing the markdown primitives per host) is a longer horizon / an **export** step, not runtime adaptation.
- **Install scope** — project-local by default (D-16); the plugin + its wiring travel with the repo.

**Scope: FULL** (owner's call, 2026-06-14) — promote the entry/router into a CI-checked integration contract across all 5 plugins + the host adapters; built incrementally:

- **✓ Increment 1 (plugins-factory 0.2.46)** — `check-integration-contract.py`, the contract as a checked property. **(H) entry/router thinness** (a command must route to a skill/agent/`bin/`/`references` AND stay < 4000 chars — the five-primitive law mechanized; calibrated on the real 37 commands, catalog passes clean) + **(I-static) advisory bundled hook** (no bundled `PreToolUse` deny — blocking is consent-wired-only, never bundled). selftest + CI.
- **Increment 2 — the behavioral hook check.** Execute each bundled `--hook` with finding-triggering input; assert exit 0 + no block-decision in stdout (catches the exit-0-but-emits-`{"decision":"block"}` escape). Executes plugin code → rides the check-mcp-liveness `--trusted-source` interlock (I-12). Per-plugin worst-case fixtures (each hook reads differently: brand/product read `content`, plugins-factory/agent-ops read disk, harness reads the path).
- **Increment 3 — `host-detect`.** Canonical `tools/host-detect/host_detect.py`, vendored (corpus-reader pattern) into harness-forge/bin (runtime: wire.py) + plugins-factory/bin (the gate's K check), sync-gated. **Process-env signals ONLY** for the live host (`CLAUDECODE` · `OPENCODE` · `GEMINI_CLI` · `HERMES_SESSION_ID` · `CODEX_SANDBOX*` medium · `AI_AGENT` low; pi has no env marker) — filesystem markers prove *installed*, never *active* (every host's config dir coexisted on the dev machine); anchored on `gh` CLI's agent-detector. Fail-safe: `unknown`.
- **Increment 4 — the (J) namespace + (K) host-awareness checks** in the gate. (J) an optional `stateNamespace` field on the two stateful plugins (`.agents/harness`, `.agents/brain`), validated against what their `bin/` actually writes (D-15 mechanized; absence = stateless). (K) every hook/MCP command is `${CLAUDE_PLUGIN_ROOT}`-rooted + each `bin/` resolves the host root with a script-relative fallback.
- **Increment 5 — the host adapter.** `wire.py` (+ the advisory hooks) consume `host-detect` to write the right config/hook format per host, or degrade gracefully under `unknown`. Full primitive portability (re-expressing the markdown primitives per host) remains an **export** step, not runtime.
